# mkingress-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存储，便于直接访问与分享。

## 项目简介

mkingress-archive 是一个纯静态页面归档仓库。仓库内收录了若干独立制作的 HTML 页面，内容涵盖信息展示、工具页面、参考文档等。所有页面均为独立文件，无后端依赖，可直接通过浏览器打开或部署至任何静态托管服务。

本项目不针对任何特定域名或网站，仅作为页面资料的集中存放与版本管理之用。

## 目录结构

```
.
├── index.html          # 仓库首页 / 导航页
├── pages/              # 归档页面存放目录
│   ├── example-page-1.html
│   ├── example-page-2.html
│   └── ...
└── assets/             # 公共资源（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：仓库入口页面，提供归档页面的索引与导航。
- `pages/`：所有归档 HTML 页面存放于此目录。
- `assets/`：存放页面共用的样式表、脚本文件及图片资源。

## 页面归档说明

- 所有页面均为静态 HTML 文件，不包含动态内容或服务器端逻辑。
- 页面之间相互独立，修改或删除某个页面不会影响其他页面的正常访问。
- 页面可能引用 `assets/` 目录下的公共资源，请确保相对路径正确。
- 每个页面均可在本地直接打开预览，也可部署至任何支持静态文件的 Web 服务器。

## 使用方式

1. 克隆本仓库到本地：
   ```
   git clone https://github.com/your-username/mkingress-archive.git
   ```
2. 直接使用浏览器打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 若需部署，将整个仓库内容上传至任意静态托管服务（如 GitHub Pages、Netlify 等）。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的导航链接。
- 修改页面：直接编辑对应 HTML 文件，注意保持资源引用路径一致。
- 删除页面：移除 `pages/` 下的文件，并同步更新 `index.html` 中的链接。
- 所有变更请遵循常规 Git 工作流，建议保留有意义的提交信息。
